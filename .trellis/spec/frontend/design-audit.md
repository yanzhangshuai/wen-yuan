# UI 设计审计清单

> 基于 `redesign-existing-projects` skill 裁剪适配，用于检查现有页面及新建页面的视觉质量。
> 同时从 `design-taste-frontend` 摘取了性能守卫与交互状态的通用规则。

---

## 使用方式

对目标页面逐条过审计项，标记 **通过 / 需修复 / 不适用**。
修复时遵循 `design-system.md` 中的项目设计系统规则。

---

## 一、排版 (Typography)

### 必查项

- [ ] **标题层级清晰**：标题通过 `font-weight`（400 → 700）+ `font-size` 建立视觉层级，不依赖单纯放大字号。
- [ ] **正文行宽受控**：中文正文每行 28–35 字，使用 `max-w-prose` 或 `max-w-[42rem]` 约束段落宽度。
- [ ] **正文行高舒适**：中文正文 `line-height` 在 1.8–2.0 范围，确保密排宋体的可读性。
- [ ] **字重梯度丰富**：不只使用 Regular(400) 和 Bold(700)，适当引入 Medium(500) 和 SemiBold(600) 建立更细腻的层级。
- [ ] **数据场景等宽**：统计数字、ID、代码使用 `font-mono`（JetBrains Mono）或启用 `font-variant-numeric: tabular-nums`，确保表格列对齐。
- [ ] **防孤行处理**：标题和短段落使用 `text-wrap: balance` 或 `text-wrap: pretty` 避免末行只剩一个字。
- [ ] **中英混排和谐**：英文片段不破坏 CJK 排版节奏；技术标识用 `font-mono` 内联。

### 反模式

- 标题使用过大负值 `letter-spacing`（CJK 字体中负 tracking 容易导致笔画粘连）。
- 全部正文使用同一 `font-weight`，缺乏视觉层次。

---

## 二、色彩与表面 (Color & Surfaces)

### 必查项

- [ ] **禁止纯黑**：不使用 `#000000`，改用 off-black（如 Zinc-950 `#09090b`）或主题变量 `var(--foreground)`。
- [ ] **颜色通过变量消费**：所有颜色使用 CSS 变量 `var(--xxx)`，禁止硬编码 hex/rgb 色值。
- [ ] **新增颜色匹配主题色相家族**：新加的强调色需与所属主题的整体色相体系协调，饱和度 < 80%。
- [ ] **阴影染色匹配背景**：`box-shadow` 使用主题背景色调的低透明度色值，而非纯黑 `rgba(0,0,0,x)`。
- [ ] **禁止 AI 渐变**：不使用紫蓝色渐变、霓虹辉光等常见 AI 生成风格。
- [ ] **背景有层次**：避免大面积纯平色块；可考虑极低 `opacity`(< 0.03) 的宣纸纹理或微噪点增加质感。
- [ ] **亮暗色突变检查**：同一页面内不出现突然的深色区块打断整体调性（除非有明确的内容分区理由）。

### 反模式

- 同一页面混用暖灰和冷灰。
- 卡片使用默认 `shadow-md` 而未染色。

---

## 三、布局 (Layout)

### 必查项

- [ ] **打破对称单调**：不是所有区块都居中对称；适当使用偏移边距、非等分栅格。
- [ ] **禁止等宽三卡片**：特性展示不使用"三等宽卡片横排"这一最常见的 AI 布局。改用交错布局、非对称网格或水平滚动。
- [ ] **全高区域安全（可滚动布局）**：`min-height` 场景使用 `min-h-dvh`（修复 iOS Safari 视口跳动）。
- [ ] **全高区域安全（固定高度画布）**：`height` / `calc(100vh - Xpx)` 等精确高度场景保持 `vh`，不替换为 `dvh`。图谱页面等桌面端全屏沉浸式布局属于此类，`dvh` 在某些桌面环境中可能与 `vh` 有微妙差异。
- [ ] **Grid 优先**：多列布局使用 CSS Grid（`grid grid-cols-*`），禁止用 flexbox 百分比算术（`w-[calc(33%-1rem)]`）。
- [ ] **容器约束**：页面内容有 `max-width` 约束（约 1200–1440px），配合 `mx-auto` 居中。
- [ ] **充足留白**：区块间距充裕，内容区不显拥挤。数据仪表盘可紧凑，但文学浏览页面应大方留白。
- [ ] **卡片组按钮底部对齐**：当多张卡片内容长度不同时，CTA 按钮固定在卡片底部形成水平线。
- [ ] **并列元素基线对齐**：定价表、对比卡、并排面板中的标题/描述/按钮在各列间对齐。
- [ ] **光学校正**：居中的图标、按钮内文字可能需要 1–2px 的光学偏移，数学居中不等于视觉居中。

### 反模式

- 使用复杂 flexbox 百分比计算代替 Grid。
- 并排卡片的特性列表起始 Y 坐标不一致。

## 四、交互与状态 (Interactivity & States)

### 必查项

- [ ] **按钮 hover 状态**：所有可点击元素有背景色变化、微位移或 `scale` 反馈。
- [ ] **按钮 active 反馈**：点击时 `scale(0.98)` 或 `translateY(1px)` 模拟物理按压感。
- [ ] **平滑过渡**：交互元素过渡时间 200–300ms，使用 `ease-out` 或自定义 `cubic-bezier`。禁止 0 时长的状态切换。
- [ ] **焦点可见**：键盘导航时所有交互元素有清晰的 focus ring。这是无障碍必需项，不可省略。
- [ ] **骨架屏加载**：列表/卡片加载时使用匹配布局尺寸的骨架屏，禁止通用圆形 spinner（页面级加载除外）。
- [ ] **空状态设计**：空数据页面有引导性的"开始使用"视图，而非空白。
- [ ] **错误状态**：表单有清晰的行内错误提示，不使用 `window.alert()`。
- [ ] **当前页导航指示**：导航中的当前页链接有视觉区分。
- [ ] **平滑滚动**：锚点跳转使用 `scroll-behavior: smooth`。

### 性能守卫（摘自 design-taste-frontend）

- [ ] **GPU 安全动画**：动画仅使用 `transform` + `opacity`，禁止动画 `top` / `left` / `width` / `height`。
- [ ] **blur 限制**：`backdrop-blur` 仅用于 `fixed` / `sticky` 元素（导航栏、弹层），禁止用于滚动容器。
- [ ] **纹理层固定**：噪点/纹理滤镜挂载在 `position: fixed; pointer-events: none` 伪元素上，不挂在滚动容器。
- [ ] **z-index 约束**：不随意使用 `z-50` 或 `z-[9999]`。z-index 仅分配给系统层级：粘性导航、弹窗、Overlay、Tooltip。
- [ ] **will-change 谨慎**：仅在正在动画的元素上使用 `will-change: transform`，动画结束后移除。

### 反模式

- 按钮只有默认态，无 hover / active / focus 样式。
- 使用通用 loading spinner 代替骨架屏。
- `backdrop-blur` 应用在大面积滚动区域导致移动端掉帧。

## 五、内容 (Content)

### 必查项

- [ ] **占位名真实化**：使用中国古典文学人物名或真实历史人名，禁止 "张三"、"李四"、"测试用户" 等无意义占位。
- [ ] **数据有机化**：避免假整数（`99.99%`、`50%`、`100`），使用有机数据（`47.2%`、`¥1,283.00`）。
- [ ] **禁止 AI 腔调**：
  - 英文：禁止 "Elevate"、"Seamless"、"Unleash"、"Next-Gen"、"Game-changer"、"Delve"。
  - 中文：禁止 "赋能"、"一站式"、"颠覆"、"沉浸式体验"、"降本增效"。
  - 使用具体、平实的语言。
- [ ] **错误提示直接**：使用明确中文错误描述（"连接失败，请重试"），禁止卖萌用语（"哎呀"、"出错啦"、"Oops!"）。
- [ ] **禁止占位文本**：不使用 Lorem Ipsum 或任何无意义填充文字。需要示例内容时使用真实的古典文学语料。
- [ ] **感叹号克制**：成功提示不用感叹号。语气自信而非激动。
- [ ] **主动语态**："无法保存更改" 而非 "更改保存失败了"。
- [ ] **日期真实**：列表中的日期应有合理的分布，不出现所有条目同一日期。
- [ ] **头像唯一**：不同用户/人物使用不同头像资源。

### 反模式

- 使用 "Acme Corp"、"Nexus"、"SmartFlow" 等西方占位品牌名。
- 所有 blog/条目使用相同日期。

---

## 六、组件模式 (Component Patterns)

### 必查项

- [ ] **卡片有理由**：卡片仅在"高度/层级传达信息层次"时使用。能用间距和分割线区分的，不加卡片容器。
- [ ] **按钮层级分明**：不总是"一个实心 + 一个描边"。适当使用文字链接或第三层级样式降低视觉噪音。
- [ ] **弹窗克制**：简单操作使用行内编辑、侧滑面板或展开区域，不为所有操作弹 Modal。
- [ ] **FAQ 不一定用手风琴**：考虑并排列表、搜索式帮助或渐进式展开。
- [ ] **shadcn/ui 组件经过定制**：不使用 shadcn 默认样式直出，通过 CSS 变量调整 radius / color / shadow 匹配项目调性。

### 反模式

- 所有信息都装进带边框 + 阴影 + 白底的卡片。
- 弹窗套弹窗的交互流程。

---

## 审计执行建议

1. **逐页审计**：每次聚焦一个页面，按上述 6 个维度逐条检查。
2. **标注优先级**：P0 = 影响可用性（无障碍、错误状态），P1 = 影响品质感，P2 = 锦上添花。
3. **截图记录**：对问题点截图并标注具体审计项编号。
4. **批量修复**：相同类型的问题一次性修复，避免逐个文件改来改去。
