# 验证证据规范

> 统一验收证据格式，避免“说通过但无证据”。

---

## 必须遵守

- 每个变更至少提供 Success/Failure/Boundary 三类证据。
- 每条证据包含：场景、步骤、期望、实际、附件（日志/截图/命令输出）。
- 禁止 `TODO/TBD/略` 作为验收结论。
- 证据应可被他人复现，不依赖口头背景。

---

## 代码/文档案例

反例：
```md
## 验证
- 已测，没问题。
```

正例：
```md
## Success Path
- 场景：章节分析成功
- 步骤：提交 chapterId=xxx
- 期望：code=ANALYZE_CHAPTER_OK
- 实际：返回 success=true, code=ANALYZE_CHAPTER_OK
- 证据：logs/analyze-success.log
```

---

## 原因

- 无结构证据无法支持回归比较与问题复盘。
- 统一模板可降低评审沟通成本，提高验收速度。

---

## 验收清单

- [ ] 三路径证据齐全
- [ ] 每条证据要素完整
- [ ] 无占位文本
- [ ] 证据可复现
