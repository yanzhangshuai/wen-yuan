## 问题

**大多数 Bug 出现在层与层的边界，而不是层内部。**

常见跨层 Bug：
- API 返回格式 A，前端却按格式 B 读取
- 数据库存 X，service 转成 Y 时丢数据
- 多层分别实现同一逻辑且实现不一致

---

## 实现跨层功能前

### 第 1 步：绘制数据流

画出数据如何流动：

```
Source → Transform → Store → Retrieve → Transform → Display
```

对每一条箭头都问：
- 数据格式是什么？
- 可能出什么问题？
- 谁负责校验？

### 第 2 步：识别边界

| 边界 | 常见问题 |
|----------|---------------|
| API ↔ Service | 类型不匹配、字段缺失 |
| Service ↔ Database | 格式转换、null 处理 |
| Backend ↔ Frontend | 序列化、日期格式 |
| Component ↔ Component | Props 结构变化 |

### 第 3 步：定义契约

对每个边界明确：
- 精确输入格式是什么？
- 精确输出格式是什么？
- 可能出现哪些错误？

---

## 常见跨层错误

### 错误 1：隐式格式假设

**Bad**：不校验就假设日期格式

**Good**：在边界处做显式格式转换

### 错误 2：校验分散

**Bad**：在多个层重复校验同一件事

**Good**：在入口层一次性完成校验

### 错误 3：抽象泄漏

**Bad**：组件知道数据库 schema 细节

**Good**：每一层只了解相邻层契约

## 跨层功能清单

实现前：
- [ ] 已映射完整数据流
- [ ] 已识别所有层边界
- [ ] 已定义每个边界的格式
- [ ] 已决定校验发生位置

实现后：
- [ ] 已用边界场景测试（null、空值、无效值）
- [ ] 已验证每个边界的错误处理
- [ ] 已确认数据往返不丢失

## 何时创建流程文档

当满足以下情况时应创建详细流程文档：
- 功能跨越 3 层以上
- 涉及多个团队协作
- 数据格式复杂
- 该功能历史上反复出 Bug
