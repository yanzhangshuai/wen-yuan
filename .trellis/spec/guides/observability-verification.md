# 可观测性验收规范

> 确保线上问题可定位、可量化、可告警。

---

## 必须遵守

- 关键请求日志至少包含：`requestId`、业务主键、`code`、耗时。
- 错误日志必须结构化，禁止仅自由文本。
- 关键失败路径应有可触发告警的稳定指标或错误码。
- 变更后需验证至少一条日志链路可从入口追踪到失败点。

---

## 代码案例

反例：
```ts
console.error("失败了");
```

正例：
```ts
console.error("analysis.failed", {
  requestId,
  chapterId,
  code: "ANALYZE_CHAPTER_FAILED",
  durationMs,
});
```

---

## 原因

- 没有结构化字段时，线上排障会退化为“全文检索猜测”。
- 稳定错误码可直接驱动告警规则，减少人工值守成本。

---

## 验收清单

- [ ] 日志字段满足最小追踪集
- [ ] 错误日志结构化且含稳定 code
- [ ] 告警规则可命中新失败路径
- [ ] 关键链路追踪可回放
