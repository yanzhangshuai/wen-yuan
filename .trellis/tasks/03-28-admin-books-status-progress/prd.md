# PRD：Admin 书籍解析状态与行操作补强

> 任务编号：`03-28-admin-books-status-progress`  
> 创建日期：2026-03-28

## 1. 背景

当前后台书库页存在两个体验问题：

1. 运营无法直观看到解析进度与阶段，容易误判“没有反应”。
2. 行操作“重试解析 / 删除书籍”缺少补充文档，后续维护成本高。

## 2. 目标

1. 在 `/admin/books` 支持查看书籍解析状态（PENDING/PROCESSING/COMPLETED/ERROR）。
2. 在 `/admin/books` 支持查看解析进度（0-100）与阶段文本（parse stage）。
3. 补充“重试解析、删除书籍、状态查询”任务文档与接口说明。

## 3. 范围

### In Scope

- 后台书籍列表状态列展示：状态徽章 + 进度条 + 阶段文本。
- `PROCESSING` 状态自动轮询状态接口。
- 行操作：重试解析、删除书籍（已实现）行为文档化。

### Out of Scope

- 新增后端接口。
- 解析任务编排策略变更。
- 书库页筛选、排序功能。

## 4. 验收标准

1. 进入 `/admin/books` 后，每行可看到状态、进度百分比与阶段。
2. 当书籍处于 `PROCESSING`，进度能自动刷新，任务结束后停止刷新。
3. 点击重试后触发 `POST /api/books/:id/analyze`，并刷新列表。
4. 点击删除后触发 `DELETE /api/books/:id`，并刷新列表。
5. 任务文档中明确三项能力的接口、行为与异常处理约定。
