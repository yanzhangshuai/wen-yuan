# 人物解析质量优化 — 过滤噪声角色 + 修复过合并

> **触发背景**：儒林外史两遍式解析结果 621 人物中，存在大量未出场角色（历史人物引用）、
> 过合并（不同人被错误合并为一人）、无具体事迹的路人角色，严重影响结果可用性。
>
> **目标**：为三类噪声角色定义明确、可复用的**过滤条件**，使系统自动剔除不应出现的条目。
> 输出数量是过滤规则作用的自然结果，**不以特定数量为目标**。
> 对于儒林外史，学术统计命名人物约 300-360 人；过滤后应比 621 显著减少，但具体数值因规则严格程度而异，不做硬性约束。

---

## 问题分类

### P0 — 过合并（身份混淆，最严重）

不同人物被错误合并为一条记录，直接导致人物关系图错乱。

**典型案例（从实际输出中提取）**：

| 错误记录 | 实际包含的不同人 | 根因 |
|---------|----------------|------|
| `姑爷` → 别名含蘧公孙、季苇萧 | 蘧公孙与季苇萧是不同人，都被叫过"姑爷" | Pass 2 消歧只看别名交集，未验证人际网络上下文 |
| `老太太/荀母` → 别名含范家老奶奶、杜老太太、鲍老太、方老太太 | 书中十几个不同家庭的母亲/老太太 | "老太太"是泛称，被误判为同一人的别名 |
| `管家/尤胡子` → 别名含宦成、邵管家、王老爹、严管家 | 不同府上的管家 | "管家"泛称跨人物合并 |
| `二相公` → 别名含匡超人、新郎 | 匡超人只是某章被叫"二相公"，被纳入婚礼场景 | 场景唯一称谓被误识别为长期别名 |

**根因分析**：
- `batchLlmDedup()` 的提示词要求 LLM 判断"这些名称是否指同一人"，但输入只有称谓列表，**没有人际关系上下文**
- 大量泛称（老太太/管家/大人/姑爷）在书中指代不同人，Union-Find 规则贪婪合并后无法区分

---

### P1 — 未出场的历史人物/引用人物被提取

在人物列表中出现大量仅在对话/典故中被提及、从未参与场景的历史人物。

**证据（实际输出中确认的问题案例）**：

```
孔夫子、孟子、曾子、屈原、董仲舒、苏轼、李太白（李太白）、
汉哀帝、董贤、秦穆公、信陵君、春申君、尧舜、
余阙、方国珍、张士诚、陈友谅、段干木、泄柳...
```

这些人物从未在小说场景中出场，仅作为文中人物引经据典时提及。

**根因分析**：
- Pass 1 (`extractChapterEntities`) 的提示词要求提取"所有人物实体"，未区分：
  - **Scene participant**（在场景中有行为/对话）
  - **Referenced figure**（被他人提及/引用，自身不出场）

---

### P2 — 无具体事迹的背景角色

书中只有一两处提及、无名无姓、或仅作为场景道具的配角被提取。

**典型案例**：
```
乌龟（词语被误识别为人物）
看茶的、走堂的、掌舵的、卖草的、卖菱小孩
又疑又聋的老妪、挑粪桶的
番子、番酋（民族泛称被识别为具名人物）
```

**根因分析**：
- 当前系统无"人物重要性"评估机制
- 在书中仅出现 1 次、且无对话/姓名的路人也被提取为正式人物

---

## 过滤条件定义（验收标准）

不以数量为目标。以下每条过滤规则独立、可验证，规则全部通过即为验收通过：

### P1 过滤条件（历史引用人物）

| 条件 | 判断依据 |
|------|----------|
| 人物在章节中**没有直接行为、对话或互动** | `participationStatus === "HISTORICAL_REFERENCE"` |
| 人物**仅出现在典故/引经据典/诗文引用**中 | Pass 1 提示词中的明确分类 |
| 人物是**真实历史先贤**（孔孟老庄等） | 历史人物黑名单兜底 |

**通过标准**：孔夫子、屈原、苏轼、尧舜等不出现在解析结果的默认视图中。

### P2 过滤条件（无事迹路人）

| 条件 | 判断依据 |
|------|----------|
| 人物**无姓名**（仅职能称谓，如"看茶的"） | `nameType === "ANONYMOUS"` 且无额外 alias |
| 人物**全书出现次数 < 2** 且无对话记录 | `mentionCount < 2 AND dialogueCount === 0` |
| canonicalName 和所有 aliases **全部在泛称黑名单中** | 无任何具名标识 |

**通过标准**：看茶的、走堂的、乌龟（骂人词误识别）等不出现在默认视图中。

### P0 过滤条件（过合并修复）

| 条件 | 判断依据 |
|------|----------|
| 两个候选的代表名称**均在泛称黑名单**中 | 阻断 Union-Find 合并 |
| 候选组的**共现关系人集合不重叠** | batchLlmDedup 输入中包含共现上下文 |

**通过标准**：蘧公孙与季苇萧不再被合并；老太太/管家不再跨家庭合并。

### 参考基线（非硬性约束）

| 指标 | 修改前 | 合理预期范围 |
|------|--------|-------------|
| 总人物数 | 621 | 显著减少，学术参考值 300-360 |
| 过合并主要角色 | ~15+ 条 | 0 |
| 历史引用人物残留占比 | ~20%+ | < 3% |
| 无事迹路人残留占比 | ~15%+ | < 5% |

---

## 非目标

- **不修改**已有历史数据（现有解析结果留待用户手动清理或重新解析）
- **不引入**书籍专属硬编码过滤规则（所有规则必须通用，通过配置/知识库调整）
- **不降低**正常出场角色的召回率（减少噪声，不减少有效角色）

## 涉及模块

| 模块 | 问题分类 | 改动性质 |
|------|---------|---------|
| Pass 1: `extractChapterEntities()` 提示词 | P1（历史引用人物）、P2（无事迹路人）| Prompt 修改 |
| Pass 2: `batchLlmDedup()` 提示词 | P0（过合并）| Prompt 修改 + 输入增强 |
| Pass 2: `buildCandidateGroups()` 规则 | P0（泛称过合并）| 算法规则修改 |
| `BOOK_VALIDATION` 后处理 | P2（低重要性过滤）| 新增过滤逻辑 |
| 提示词中"出场人物"定义约束 | P1 | Prompt 修改 |
