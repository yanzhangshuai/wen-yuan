## 2. 工作包与条目

## WP-A 图谱能力补齐（P0）

### FG-01 图谱导出 PNG/SVG

- 判定：未实现  
- 要求：支持导出 PNG / SVG / JSON  
- 交付：
  - 实现 `handleExport("png" | "svg")`；
  - 保持 JSON 导出兼容；
  - 导出失败提供可见反馈（toast）。
- 验收：
  - 点击导出 PNG/SVG 均可下载并可打开；
  - 节点、边、标签与当前视图一致；
  - 不影响既有 JSON 导出。

### FG-02 布局 tree 真正可用

- 判定：未实现  
- 要求：工具栏“层级树”可切换并生效  
- 交付：
  - `layoutMode === "tree"` 时应用树布局（或明确的层级布局算法）；
  - 切换布局后保持缩放/拖拽交互一致。
- 验收：
  - `force/radial/tree` 三种布局可互切；
  - tree 不是 radial 视觉复用；
  - 切换不触发异常闪烁。

### FG-03 节点形状按实体类型渲染

- 判定：未实现  
- 要求：人物圆形、地点菱形、组织六边形  
- 交付：
  - 基于 `personaType`（或等价字段）映射 `nodePath`；
  - 无类型数据时有安全兜底。
- 验收：
  - 三类节点形状可区分；
  - 旧数据不崩溃；
  - 图例与视觉一致。

### FG-04 拖拽后布局持久化

- 判定：未实现  
- 要求：拖拽后调用 `PATCH /api/graphs/:id/layout` 持久化  
- 交付：
  - 节点拖拽结束后节流/防抖保存；
  - 刷新页面后位置可恢复；
  - 保存失败提示用户并可重试。
- 验收：
  - 拖拽节点 -> 刷新 -> 坐标保持；
  - 批量拖拽不会造成请求风暴；
  - 无权限/失败路径有友好提示。

### FG-05 图谱内联校对链路补全

- 判定：未实现（占位）  
- 要求：右键节点支持编辑、合并、删除与状态流转  
- 交付：
  - `onEdit/onMerge/onDelete` 从占位变为真实行为；
  - 编辑态可修改文档要求字段；
  - 状态操作支持 `DRAFT -> VERIFIED/REJECTED`。
- 验收：
  - 右键动作均可触发并完成请求；
  - 操作后图谱与侧栏状态同步；
  - 失败反馈可见。

### FG-06 原文面板“上一处/下一处”证据导航

- 判定：未实现  
- 要求：支持证据点前后跳转  
- 交付：
  - `TextReaderPanel` 调用处传入 `onPrev/onNext`；
  - 与证据索引状态联动；
  - 边界场景（首条/末条）处理清晰。
- 验收：
  - 按钮可见且可用；
  - 跳转后高亮与滚动定位正确；
  - 无证据时按钮隐藏或禁用。

## WP-B 审核与原文联动（P0/P1）

### FG-07 草稿条目点击打开原文高亮

- 判定：未实现  
- 要求：点击任意草稿可打开原文面板并高亮引用段落  
- 交付：
  - Persona/Relationship/Biography 三类草稿行支持“点击打开原文”；
  - 与现有编辑/确认/拒绝动作不冲突；
  - 高亮段落由证据映射得到。
- 验收：
  - 三类草稿均可打开对应章节原文；
  - 高亮位置准确；
  - 行为不影响批量操作。

### FG-08 合并建议补齐证据入口与原文对照

- 判定：部分实现  
- 要求：展示原文证据入口，支持双方详情与原文对照  
- 交付：
  - 合并建议卡增加证据入口；
  - `EntityMergeTool` 增加原文证据面板或抽屉；
  - 明确“是否涉及已审核人物”的标记。
- 验收：
  - 每条建议可追溯证据；
  - 审核员可在一个流里看双方信息+原文；
  - 不影响已有接受/拒绝/稍后操作。

## 3. 回归与验证清单

## 3.1 自动化

- `pnpm lint`
- `pnpm typecheck`
- 图谱与审核相关测试（若已有）全部通过

## 3.2 手测（必须）

1. 图谱页：导出、布局切换、节点拖拽持久化、右键校对、证据前后跳转  
2. 审核页：点击草稿开原文、合并建议看证据并完成合并决策  
3. 导入页：Step3 修改标题、合并章节后启动解析  
4. 前台书库：非完成书籍无 Badge 且不可点击  
5. Admin：退出后会话失效  
6. SEO/Router：graph metadata、全局 OG、各段 error.tsx 生效
