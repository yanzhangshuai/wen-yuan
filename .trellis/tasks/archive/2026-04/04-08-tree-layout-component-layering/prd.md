# implement: tree 布局改造（连通分量分层 + 稳定选根 + 非树节点单独泳道）

## Goal

将当前 `tree` 布局从“单根 BFS 伪层级”升级为“可解释、可读的分量级分层布局”，避免节点塌缩到同一层，提升路径查找后的可视化可用性。

## Background

- 当前 `tree` 布局在 `src/components/graph/force-graph.tsx` 中使用“影响力最高节点”作为唯一根节点。
- 单根 BFS 后，未触达节点会被统一放入末层，导致多连通分量数据出现“大量节点挤在一层”。
- UI 模式文案为“层级树”，但算法语义更接近“单根可达分层”，存在认知偏差。

## Scope

### In Scope

- 前端 `tree` 布局算法改造为“连通分量分层”。
- 每个连通分量内采用稳定选根策略（`influence -> degree -> id`）。
- 非树语义节点（无法参与分层或未触达）进入单独泳道/网格区，不与主层级混排。
- 保持与现有主题、高亮、路径查找联动兼容。

### Out of Scope

- 不改后端关系存储结构。
- 不新增家谱本体模型或关系类型枚举治理。
- 不改 `force/radial` 布局行为。

## Requirements

- R1: `tree` 布局必须先做连通分量划分，再对每个分量独立分层。
- R2: 根节点选择必须稳定且可重复，排序规则固定为 `influence desc -> degree desc -> id asc`。
- R3: 主层级区与“非树节点泳道”视觉区隔清晰，避免全部节点回落到同一层。
- R4: 布局完成后保留自动 fit 能力，保证主要内容首屏可读。
- R5: 布局结果需兼容节点 hover/active/path 高亮，且主题切换下对比度可用。

## Acceptance Criteria

- [ ] 在多连通分量数据下，节点不再集中到单一横带。
- [ ] 同一输入数据重复渲染时，tree 布局结果稳定（根选择和层级顺序一致）。
- [ ] 非树节点不与主层级节点混排，展示在单独泳道/网格区。
- [ ] 路径查找成功后，tree 模式下节点和边高亮仍正确，且自动适配到合适视野。
- [ ] 浅色/深色主题下，主层级与泳道区、普通态与高亮态均可清晰区分。

## Technical Notes

- 主要改造文件：`src/components/graph/force-graph.tsx`
- 建议新增小型纯函数（组件划分、稳定选根、分层位置计算）以提升可测性。
- 若需要文案降级（如“分层视图”），同步 `src/components/graph/graph-toolbar.tsx`。

## Risks & Mitigation

- 风险：层级计算复杂度上升导致首帧变慢。
- 缓解：限制每轮布局计算在当前过滤后的子图执行，并复用现有坐标缓存。

- 风险：泳道区与主图冲突导致视觉拥挤。
- 缓解：按容器尺寸动态计算泳道偏移与列数，必要时限制标签显示密度。
