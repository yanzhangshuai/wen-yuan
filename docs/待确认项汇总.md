## 已确认决策清单

### D1. "牛浦" 归属 ✅ 已确认

- **结论**: 牛布衣与牛浦郎是两个独立人物。牛布衣真实存在，后已去世；牛浦郎以"牛布衣"之名招摇过市。凡冒名期间发生的经历、行为、事件，均归属"牛浦郎"这一真实行为人，不归属牛布衣。
- **操作**: `classical-names.ts` 中 `牛布衣` aliases 移除 `牛浦郎` 及其他牛浦相关错误映射；seed 数据新增独立条目 `{ canonicalName: "牛浦郎", aliases: ["牛浦"] }`，并通过 `AliasMapping(source=IMPERSONATION)` 记录其冒用"牛布衣"身份的章节范围。
- **关联任务**: deprecate-classical-names

### D2. 泛称存储 ✅ 已确认

- **结论**: 所有泛称词表统一进入知识库 DB 管理，硬编码全部移除。不保留"先硬编码再迁移"的过渡策略。
- **操作**: `lexicon.ts` 中 `SAFETY_GENERIC_TITLES`、`DEFAULT_GENERIC_TITLES`、`HARD_BLOCK_SUFFIXES`、`DEFAULT_SOFT_BLOCK_SUFFIXES`、`UNIVERSAL_TITLE_STEMS`、`DEFAULT_POSITION_STEMS` 全部移除，代码层仅保留从 DB 加载的通道。
- **影响**: Wave1 filter-hardening 与 Wave2 kb-schema-extend 合并为同一序列——先写入 DB，再改代码引用。

### D3. 自动合并阈值 ✅ 已确认

- **结论**: 仅 Tier 1（conf = 1.0，精确名称匹配）自动合并。其余所有层级一律写入 `merge_suggestions` 表、status=PENDING，等待人工确认。
- **约束**: 不允许扩大自动合并范围来换取表面准确率。

### D4. 历史人物库范围 ✅ 已确认

- **结论**: 构建通用中国历史人物库（500+ 条），不局限于儒林外史单部作品。
- **数据分类**: EMPEROR / SAGE / POET / GENERAL / MYTHICAL / STATESMAN

### D5. 历史人物数据产出方式 ✅ 已确认

- **结论**: LLM 生成 + 人工抽检，持续迭代优化提示词内容，逐步提升生成准确性。
- **流程**: LLM 按朝代/类别批量生成 → 人工抽检 → 修正提示词 → 再生成 → 入库审核

### D6. 字号映射数据来源 ✅ 已确认

- **结论**: 开发者主导 + LLM 辅助。所有书籍类型均采用此模式推进。
- **流程**: 开发者提供核心样本 → LLM 按样本格式扩充 → 人工校验 → 入库

### D7. dynamicTitleResolution 开启时机 ✅ 已确认

- **结论**: 修复后开启。所有新增能力（dynamicTitleResolution、llmTitleArbitration、知识库驱动过滤等）统一标记为"修复后开启"，不在修复完成前全量启用。

### D8. Book.genre 字段处理 ✅ 已确认

- **结论**: 直接废弃删除，不做数据迁移。Prisma schema 中移除 `genre` 字段，所有引用改为 `bookTypeId` → `BookType.key`。
- **操作**: 新增 Prisma migration 删除 `genre` 列；代码中所有 `book.genre` 引用替换。

### D9. 正则安全策略 ✅ 已确认

- **结论**: 强制执行以下三条规则：
  1. 编译时 100ms 超时
  2. 正则长度 ≤ 200 字符
  3. 禁止嵌套量词（`(a+)+`、`(a*)*` 等模式）
- **操作**: `NamePatternRule` 入库时做校验，编译时 wrap try-catch + setTimeout。

### D10. 金标准标注 ✅ 已确认

- **结论**: 开发者基于审计报告手工标注 50-80 条核心角色样本，作为评估与回归基线。

### D11. GENRE_PRESETS / 旧结构处理 ✅ 已确认

- **结论**: 不再需要的旧字段/旧配置/旧兼容逻辑直接删除，不做迁移。包括：
  - `GENRE_PRESETS` — 直接删除
  - `classical-names.ts` — 数据迁移到 DB 后直接删除
  - `Book.genre` — 直接删除

### D12. 缓存失效策略 ✅ 已确认

## 业务规则补充确认（D13，本轮新增）

### D13. 历史人物提取规则 ✅ 已确认

- **结论**: 历史人物不是一律不提取，也不是一律提取。规则如下：
  - 如果历史人物在书内有实际经历、行为、对话、事件参与（如朱元璋以"吴王"身份出现在《儒林外史》第一回，有具体行为），**需要被提取**
  - 主实体使用真名存储（如：朱元璋），书内称谓/阶段性身份/别名记录为 alias（如：吴王、太祖皇帝）
  - 仅纯提及、引用、典故性出现、背景提及，而无书内实际经历的人物，**不提取为主角色**
- **操作**: `HistoricalFigureEntry` 表不再是简单黑名单，而是"已知历史人物标记库"。解析链路对命中项需进一步判断"书内是否有实际参与"后决定是否保留。
