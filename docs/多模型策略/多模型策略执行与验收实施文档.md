## 3. 目标与非目标

### 3.1 目标

1. 固定 20 章金标集，建立阶段级 A/B 对比能力。
2. 统一输出 4 个核心指标：
   - JSON 成功率
   - 人物 F1 / 关系 F1
   - 每万字成本
   - 吞吐时间
3. 基于客观结果产出最终 `stage model map`。
4. 将验收流程做成 `codex` 可执行命令链，支持一键判定是否达标。

### 3.2 非目标

1. 不改动业务主流程的语义契约（提取字段、入库模型、响应结构）。
2. 不引入与当前业务无关的新模块（例如 OCR 处理）。
3. 不以“主观模型印象”直接拍板最终模型。

## 6. 指标定义与计算口径（验收核心）

### 6.1 JSON 成功率

- 定义：`JSON 成功解析且通过 schema 校验的调用数 / 总调用数`。
- 统计粒度：按阶段、按模型、按全实验。
- 门槛建议：`>= 98%`（Phase2 必须达标）。

### 6.2 人物 F1 / 关系 F1

- 定义：基于 20 章金标计算精确率、召回率、F1。
- 人物匹配：标准名 + alias 映射后比对。
- 关系匹配：`source,target,type` 三元组。
- 门槛建议：
  - 不低于当前生产基线
  - 目标提升 `>= 1.5%`

### 6.3 每万字成本

- 定义：`(实验总成本 / 实验总字数) * 10000`。
- 数据来源：`analysis_phase_logs` + 模型价格表。
- 门槛建议：相对基线下降 `>= 20%`，若 F1 显著提升可豁免。

### 6.4 吞吐时间

- 定义：单万字处理耗时 + 全实验 wall-clock。
- 门槛建议：相对基线缩短 `>= 15%`。

## 7. 三路径验收方案（可执行）

### 7.1 Success Path

- 场景：完整跑通金标实验并得到可用 stage map。
- 执行：
  1. 校验金标。
  2. 跑 Phase1/2/5/全书验证 A/B。
  3. 计算指标并跑 gate 判定。
- 期望：`gate.result.json` 为 `PASS`。

### 7.2 Failure Path

- 场景：故意注入异常输入，验证错误处理稳定。
- 执行：
  1. 使用非法 goldset 行触发 schema 失败。
  2. 使用无效 phase 参数触发 CLI 参数错误。
  3. 模拟模型不可达，验证 fallback/错误码。
- 期望：返回稳定错误码、错误信息可读、流程不中断或按预期失败。

### 7.3 Boundary Path

- 场景：边界与极端输入。
- 执行：
  1. 空 chapter list。
  2. 超长章节（触发 Phase1 长输入保护）。
  3. 最小样本量（1 章）与最大样本量（20 章）对比。
- 期望：系统平稳处理，无崩溃、无静默吞错。

## 8. Codex 可执行验收命令（推荐一键链路）

以下命令链作为 `codex5.3` 的标准执行顺序。

```bash
# 0) 基础质量门禁
pnpm lint
pnpm type-check
pnpm test

# 1) 金标校验
pnpm ts-node scripts/eval/validate-goldset.ts \
  --schema data/eval/goldset.schema.json \
  --input data/eval/goldset.v1.jsonl

# 2) Phase1 A/B
pnpm ts-node scripts/eval/run-stage-ab.ts \
  --phase ROSTER_DISCOVERY \
  --book-id <BOOK_ID> \
  --chapter-list data/eval/chapters.20.json \
  --candidate-set config/model-candidates.v1.json \
  --experiment-tag phase1_ab_v1

# 3) Phase2 A/B
pnpm ts-node scripts/eval/run-stage-ab.ts \
  --phase CHUNK_EXTRACTION \
  --book-id <BOOK_ID> \
  --chapter-list data/eval/chapters.20.json \
  --candidate-set config/model-candidates.v1.json \
  --experiment-tag phase2_ab_v1

# 4) Phase5 A/B
pnpm ts-node scripts/eval/run-stage-ab.ts \
  --phase TITLE_RESOLUTION \
  --book-id <BOOK_ID> \
  --chapter-list data/eval/chapters.20.json \
  --candidate-set config/model-candidates.v1.json \
  --experiment-tag phase5_ab_v1

# 5) 全书验证 A/B
pnpm ts-node scripts/eval/run-stage-ab.ts \
  --phase BOOK_VALIDATION \
  --book-id <BOOK_ID> \
  --chapter-list data/eval/chapters.20.json \
  --candidate-set config/model-candidates.v1.json \
  --experiment-tag bookval_ab_v1

# 6) 汇总指标
pnpm ts-node scripts/eval/compute-metrics.ts \
  --experiments phase1_ab_v1,phase2_ab_v1,phase5_ab_v1,bookval_ab_v1 \
  --goldset data/eval/goldset.v1.jsonl \
  --output docs/eval/metrics.summary.json

# 7) 门禁判定（自动 PASS/FAIL）
pnpm ts-node scripts/eval/check-gate.ts \
  --metrics docs/eval/metrics.summary.json \
  --baseline docs/eval/baseline.metrics.json \
  --output docs/eval/gate.result.json
```
