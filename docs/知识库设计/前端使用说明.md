# 知识库管理后台 — 前端使用说明

> 文档版本：1.0  
> 最后更新：2026-04-10  
> 适用对象：拥有管理后台访问权限的操作人员

---

## 总览

知识库管理后台提供 7 个管理页面，每个页面对应一类知识资源，共同支撑古典文学人物解析系统的精准分析。

| 页面路径 | 中文名 | 核心用途 |
|---------|--------|---------|
| `/admin/knowledge-base/book-types` | 书籍类型管理 | 增删书籍类型、配置各类型的 NER 词表规则 |
| `/admin/knowledge-base/alias-packs` | 人物别名知识包 | 管理人物别名映射包，支持 AI 批量生成与人工审核 |
| `/admin/knowledge-base/surnames` | 姓氏识别库 | 维护中文姓氏表，支持复姓优先匹配 |
| `/admin/knowledge-base/title-filters` | 称谓过滤词库 | 配置哪些通用称谓不应被识别为独立人物名 |
| `/admin/knowledge-base/prompt-templates` | AI 提示词模板库 | 管理 AI 各分析阶段的提示词，支持版本历史与回滚 |
| `/admin/knowledge-base/ner-rules` | 实体识别规则库 | 管理注入提示词的实体提取规则，可按书籍类型扩展 |
| `/admin/knowledge-base/change-logs` | 知识变更记录 | 查看全部知识库资源的历史变更操作 |

## 一、书籍类型管理 `/admin/knowledge-base/book-types`

### 功能说明

书籍类型是知识库组织的核心维度。每个书籍类型对应：
1. 一个唯一的 `key`（被解析流水线引用，**一旦设定不要随意修改**）
2. NER 词表调谐配置（`presetConfig`），控制该书籍类型下哪些称谓会被特殊处理

### 操作步骤

**查看书籍类型列表**  
- 列表显示：类型键（key）/ 展示名（name）/ 关联知识包数 / 关联书籍数 / 启用状态
- 点击列表行展开详情

**创建新书籍类型**  
1. 点击右上角【添加书籍类型】按钮
2. 填写表单：
   - `key`（必填）：系统内部唯一标识，建议用中文简短名称，如"仙侠"
   - `name`（必填）：前端展示名称（可与 key 不同）
   - `sortOrder`：列表排序编号（数字越小越靠前）
   - `description`：类型描述（可选）
3. 展开【NER 配置编辑器】，按需配置（见下方字段说明）
4. 点击【保存】

**NER 配置编辑器字段说明**

| 字段 | 含义 | 示例 |
|------|------|------|
| `exemptGenericTitles` | 在此书籍类型下，这些词**不会**被当作称谓过滤掉，允许识别为具体人物 | 武侠：`["掌门","帮主","盟主"]` |
| `additionalTitlePatterns` | 补充职位/头衔识别词，提升该书籍类型的职位识别召回率 | `["太守","刺史","都督"]` |
| `additionalPositionPatterns` | 补充官职识别词 | `["司马","司徒"]` |
| `softRelationalSuffixes` | 这些后缀作为亲属关系处理，不独立创建人物实体 | 家族世情：`["哥哥","姐姐"]` |

> ⚠ **重要**：`presetConfig` 修改后立即生效，下一次分析任务启动时会加载新配置。已完成的分析不受影响。

**编辑书籍类型**  
- 点击列表行的【编辑】按钮，进入编辑模式
- 注意：修改 `key` 会导致关联书籍的引用失效，**请勿修改已有书籍使用的书籍类型 key**

**删除书籍类型**  
- 若该书籍类型下有关联书籍，系统不允许删除，需先在书籍管理页修改书籍类型
- 若仅希望隐藏（不在导入页下拉框中显示），改用【停用】操作

## 二、人物别名知识包 `/admin/knowledge-base/alias-packs`

### 功能说明

通过"别名知识包"组织人物别名信息：
- **书籍类型级包（GENRE）**：同类书籍类型的所有书籍自动继承
- **书籍专属包（BOOK）**：仅对手动挂载的书籍生效，优先级更高

### 左栏：知识包列表操作

**创建知识包**  
1. 点击【创建知识包】
2. 选择 scope：
   - `GENRE`（书籍类型级）：需选择关联书籍类型
   - `BOOK`（书籍专属）：可不关联书籍类型，需后续在书籍管理页手动挂载
3. 填写包名称和描述
4. 点击【保存】

**知识包统计角标**  
- 红色数字角标：该包中待审核（PENDING）条目数
- 每次 LLM 生成或导入后会自动增加待审核计数

### 右栏：条目管理操作

选中左栏的知识包后，右栏展示该包的条目详情。

**筛选条目**  
顶部筛选栏支持：
- 状态筛选：全部 / 待审核 / 已验证 / 已拒绝
- 关键词搜索：搜索 canonicalName 或 aliases

**添加单条条目**  
1. 点击【添加条目】
2. 填写：标准名（canonicalName）、别名列表（alias chip 组，按回车确认一个别名）
3. 选择条目类型（CHARACTER / LOCATION / ORGANIZATION）
4. 点击【保存】，条目初始状态为 PENDING（待审核）

**编辑条目**  
单击条目行展开内联编辑器：
- 修改标准名
- 添加/删除别名（chip 样式，点击 × 删除，输入框内按回车添加）
- 点击【保存更改】

**审核条目**  
- 待审核行右侧有快速操作按钮：✓（通过）/ ✗（拒绝）
- 拒绝时需填写原因（弹出小型输入框）
- 可勾选多行后点击工具栏【批量审核通过】或【批量拒绝】
- ⚠ 只有状态为 **VERIFIED（已验证）** 的条目才参与解析流水线

**各状态说明**

| 状态 | 含义 | 是否参与解析 |
|------|------|------------|
| PENDING（待审核） | 新建 / LLM 生成 / 导入后的初始状态 | ❌ 不参与 |
| VERIFIED（已验证） | 人工审核通过 | ✅ 参与 |
| REJECTED（已拒绝） | 审核拒绝 | ❌ 不参与 |

### LLM 批量生成操作

1. 选中知识包，点击右上角【LLM 生成】
2. 在弹窗中配置：
   - 目标书籍（可选）：选择书籍可让 AI 基于该书理解生成
   - 生成模型：选择可用的 AI 模型
   - 期望人物数：建议 30-100 之间，过大可能降低精度
3. 切换【提示词预览】查看将发送给 AI 的实际内容（可展开编辑自定义）
4. 确认后点击【开始生成】
5. 生成完成后自动跳转到**审核界面**（不直接写入库）

**审核生成结果**  
- 系统自动按置信度排序，低于 0.5 的条目预标"拒绝"
- 与已有条目别名重叠的显示橙色警告
- 勾选需要保留的条目，点击【保存选中条目】（写入 PENDING 状态）
- 未保存的条目会被丢弃
- 保存后在条目列表的"待审核"视图中继续审核

### 导入操作

1. 点击工具栏【导入】，选择格式（JSON 或 CSV）
2. 拖拽或点击上传文件
3. 查看解析预览（成功条数、格式错误说明）
4. 选择导入后状态：
   - PENDING（推荐）：导入后需人工审核
   - VERIFIED：确认数据来源可信时使用（如从官方文学数据库导入）
5. 点击【确认导入】

**JSON 格式模板**：
```json
{
  "entries": [
    {
      "canonicalName": "关羽",
      "aliases": ["关云长", "云长", "关公", "美髯公"],
      "entryType": "CHARACTER",
      "notes": "蜀汉五虎上将"
    }
  ]
}
```

**CSV 格式说明**：
- 首行必须为字段名：`canonicalName,aliases,entryType,notes`
- `aliases` 用 `|` 分隔（如 `关云长|云长|关公`）
- 仅 `canonicalName` 为必填字段

### 导出操作

点击工具栏【导出】：
- 默认导出 JSON 格式
- 导出文件名：`{包名}_v{版本}_{日期}.json`
- 仅导出该包的 VERIFIED 条目（REJECTED / PENDING 不导出）
- 如需导出全部状态，在导出选项中勾选

## 三、姓氏识别库 `/admin/knowledge-base/surnames`

### 功能说明

系统通过读取姓氏库来识别人物称谓中的姓氏部分（如"范举人"→识别姓氏"范"→匹配到范进）。复姓（欧阳/司马）必须高优先级，否则会被截断为单字姓。

### 操作说明

**查看与筛选**  
- 顶部切换[复姓]/[单姓]/[全部]快速筛选
- 支持关键词搜索

**添加姓氏**  
1. 点击【添加姓氏】
2. 输入姓氏内容
3. 勾选"是否复姓"
4. 设置优先级（复姓建议 10，特殊罕见姓可设更高）
5. 可选关联书籍类型（表示此姓主要出现于某类型作品）

> ⚠ **复姓必须勾选"是否复姓"并设置高优先级**，否则系统可能将"欧阳"误识别为单姓"欧"

**批量导入**  
点击【导入】，支持：
- 换行符分隔的纯姓氏文本（一行一个）
- 系统自动判断是否为复姓（长度>=2 且不在已知单姓集合中）
- 导入前可预览识别结果并手动修正

**测试工具**  
页面底部内置测试面板：
- 输入任意人名，点击【测试提取】
- 显示提取结果：提取到的姓氏、匹配类型（复姓/单姓）、优先级
- 用于验证新添加的姓氏是否生效

## 四、称谓过滤词库 `/admin/knowledge-base/title-filters`

### 功能说明

系统使用称谓过滤词库过滤不应视为独立人物的通用称谓。例如"老爷"通常不指特定人，但在家族类小说中可能指具体角色。

### 称谓级别（Tier）说明

| 级别 | 含义 | 能否删除 | 能否豁免 |
|------|------|---------|---------|
| 🔒 安全泛称（SAFETY） | 在任何情况下都不应个体化（如"他""此人""父亲"） | 否，只能停用 | 否 |
| 默认泛称（DEFAULT） | 默认视为泛称，但可在特定书籍类型下豁免 | 是 | 是 |

### 添加称谓

1. 点击【添加称谓】
2. 输入称谓文本
3. 选择级别：DEFAULT（大多数情况）或 SAFETY
4. 设置书籍类型豁免：在"书籍类型豁免"多选框中选择该称谓不应被过滤的书籍类型
5. 保存

**书籍类型豁免逻辑示例：**
- "掌门"在武侠书籍类型下指具体人物 → 豁免书籍类型选"武侠"
- "大王"在神魔小说中指具体角色 → 豁免书籍类型选"神魔小说"

### 测试工具

页面底部：
- 输入称谓 + 选择书籍类型
- 返回：在此书籍类型下是否视为过滤称谓，以及原因说明

## 五、AI 提示词模板库 `/admin/knowledge-base/prompt-templates`

### 功能说明

系统有 7 个分析模板，每个对应解析流水线的一个阶段。提示词直接影响 AI 分析质量，此页面允许在不修改代码的情况下调整提示词并跟踪历史。

| 模板 slug | 名称 | 影响阶段 |
|----------|------|---------|
| `INDEPENDENT_EXTRACTION` | Pass 1 独立提取 | 章节人物初次提取（核心，改动影响最大） |
| `ENTITY_RESOLUTION` | Pass 2 实体消歧 | 全书人物去重合并 |
| `TITLE_RESOLUTION` | 称号溯源 | 别名/字号追溯标准名 |
| `TITLE_ARBITRATION` | 称号仲裁 | 灰区称谓的 LLM 最终判定 |
| `CHAPTER_VALIDATION` | 章节质量校验 | 验证章节分析结果 |
| `BOOK_VALIDATION` | 全书质量校验 | 验证全书数据一致性 |
| `ROSTER_DISCOVERY` | 名册发现（已废弃） | 两遍式架构下不使用 |

### 编辑提示词

1. 点击模板行的【编辑】
2. 进入版本编辑器，分为【系统提示词】和【用户提示词】两个标签页
3. 编辑内容（支持 Markdown 语法预览）
4. 可用的占位符（编辑器会自动提示）：

| 占位符 | 替换内容 |
|--------|---------|
| `{bookTitle}` | 当前书名 |
| `{chapterNo}` | 章节编号 |
| `{chapterTitle}` | 章节标题 |
| `{content}` | 正文剧情片段 |
| `{genericTitles}` | 当前书籍类型的有效过滤称谓列表 |
| `{profiles}` | 已建档人物摘要 |
| `{genreKey}` | 当前书籍类型键 |

5. 填写本次修改说明（`changeNote`，建议必填，方便回溯）
6. 点击【保存为新版本】（不会覆盖旧版本）

### 激活版本

- 每个模板同时只有一个**激活版本**（解析时使用）
- 版本历史列表中，点击某个版本的【激活】可切换
- 激活操作会自动记录审计日志

### 预览功能

在版本详情页，点击【预览】：
1. 系统展示填充了示例数据的完整提示词
2. 显示预估 token 数（帮助评估成本）
3. **不执行任何 AI 调用**，纯本地渲染

### 版本对比（Diff）

选择两个版本后点击【对比差异】，以 diff 格式展示：
- 红色：删除的内容
- 绿色：新增的内容

> ⚠ **警告**：`INDEPENDENT_EXTRACTION`（Pass 1）是最核心的模板，对其的任何变更都可能显著影响所有书籍的解析质量。建议修改后先对已验证的评估样本集运行小范围测试，再激活新版本应用于生产。

## 六、实体识别规则库 `/admin/knowledge-base/ner-rules`

### 功能说明

实体识别规则是注入提示词的自然语言指令列表，告诉 AI 如何提取实体。规则分两类：
- **实体规则（ENTITY）**：控制人物实体的提取行为
- **关系规则（RELATIONSHIP）**：控制人物关系的提取行为

### 操作说明

**切换规则类型**  
顶部标签页切换【实体规则】/【关系规则】

**查看规则**  
列表显示规则序号、内容预览、生效范围（通用/某书籍类型）、启用状态。  
规则按 `sortOrder` 排列，序号即最终注入提示词时的顺序。

**添加规则**  
1. 点击【添加规则】
2. 选择规则类型
3. 输入规则正文（中文自然语言，直接作为 prompt 中的列表项）
4. 选择**生效范围**：
   - "通用"：所有书籍类型都加载此规则
   - 选择某书籍类型：仅该书籍类型会额外叠加此规则
5. 保存

**调整顺序**  
行尾的【↑↓】按钮拖动调整排序。顺序影响 AI 解读规则的优先级（靠前的规则权重更高）。

**停用规则（不删除）**  
点击【停用】将规则标记为停用，但保留记录方便日后重新启用或对照对比。

**合并预览**  
页面底部的"合并预览"工具：
- 选择书籍类型，点击【预览注入列表】
- 展示在该书籍类型下，最终注入提示词的完整规则列表（通用 + 类型专属）

## 七、知识变更记录 `/admin/knowledge-base/change-logs`

### 功能说明

记录所有知识库的变更历史，无需手动操作，系统自动记录。

### 筛选查询

| 筛选维度 | 支持值 |
|---------|-------|
| 对象类型 | 知识条目 / 别名知识包 / 书籍类型 / 姓氏 / 过滤称谓 / 提示词模板 / 识别规则 |
| 操作类型 | 创建 / 修改 / 删除 / 审核通过 / 审核拒绝 / 激活 / 停用 / 导入 / 生成 |
| 时间范围 | 起止日期选择 |
| 操作人 | 输入用户 ID 或名称 |

### 查看变更详情

点击列表行，展开变更前/后的 JSON 对比：
- 红色：修改前的值
- 绿色：修改后的值
- 用于精确排查"某次修改导致解析变差"的根因

### 典型使用场景

1. **分析质量下降排查**：查看最近 3 天内的提示词激活、规则修改记录
2. **数据来源追踪**：查看某条知识条目是 LLM 生成还是手动录入，历次修改记录
3. **合规审计**：了解特定操作人的所有变更历史

## 附录：各页面权限要求

> 所有知识库管理页面均要求管理员（admin）角色登录，普通用户无权访问。

| 页面 | 读取 | 写入 | 删除 | 备注 |
|------|------|------|------|------|
| 书籍类型管理 | admin | admin | admin | 删除需先迁移关联书籍 |
| 人物别名知识包 | admin | admin | admin | SAFETY 过滤称谓不可删除 |
| 姓氏识别库 | admin | admin | admin | — |
| 称谓过滤词库 | admin | admin | admin | SAFETY 级别只能停用 |
| AI 提示词模板库 | admin | admin | — | 只支持新增版本，不支持删除历史版本 |
| 实体识别规则库 | admin | admin | admin | 建议停用而非删除 |
| 知识变更记录 | admin | 系统自动 | — | 日志只读，不可删除 |

## 附录：常见操作快速参考

| 场景 | 操作路径 |
|------|---------|
| 书籍类型未覆盖，需新增类型 | `/admin/knowledge-base/book-types` → 添加书籍类型 → 配置 NER presetConfig |
