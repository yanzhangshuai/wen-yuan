# 知识库初始化操作手册

> 文档版本：1.1  
> 最后更新：2026-04-11  
> 适用阶段：首次部署知识库功能时的数据初始化  
> 前置条件：已完成知识库 Schema 迁移（Phase 6，见主设计文档第 20 章）

---

## 概述

本手册指导在完成数据库 Schema 迁移后，将系统中已有的硬编码知识迁移到数据库，并初始化所有必要的知识库表数据。

**初始化包含以下内容：**

| 数据类型 | 数据来源 | 条目规模 |
|---------|---------|---------|
| 书籍类型表（Genre） | `pipeline.ts` → `GENRE_PRESETS` | 7 个书籍类型 |
| 人物别名知识包（KnowledgePack + KnowledgeEntry） | `classical-names.ts` → `GENRE_CLASSICAL_NAMES` | 5 个包，~131 条 |
| 姓氏库（SurnameEntry） | `lexicon.ts` → `CHINESE_SURNAME_LIST` | ~148 条（含 18 复姓）|
| 泛化称谓库（GenericTitleEntry） | `lexicon.ts` → `SAFETY_GENERIC_TITLES` + `DEFAULT_GENERIC_TITLES` | ~50 条 |
| NER 提取规则（ExtractionRule） | `lexicon.ts` → `ENTITY_EXTRACTION_RULES` + `RELATIONSHIP_EXTRACTION_RULES` | 12 条 |
| 提示词模板（PromptTemplate + PromptTemplateVersion） | `prompts.ts` + 运行时校验链路 → 8 个模板槽 | 8 个模板，各 1 个可执行基线版本 |

// prisma/seed-knowledge.ts (部分6: 提示词模板)
export async function seedPromptTemplates(prisma: PrismaClient) {
  for (const slot of PROMPT_TEMPLATE_BASELINES) {
    const template = await prisma.promptTemplate.upsert({
      where: { slug: slot.slug },
      create: {
        slug: slot.slug,
        name: slot.name,
        description: slot.description,
        codeRef: slot.codeRef,
        isActive: slot.isActive ?? true,
      },
      update: {
        name: slot.name,
        description: slot.description,
        codeRef: slot.codeRef,
        isActive: slot.isActive ?? true,
      }
    });

    const latestVersion = await prisma.promptTemplateVersion.findFirst({
      where: { templateId: template.id },
      orderBy: { versionNo: 'desc' }
    });

    if (!latestVersion || latestVersion.systemPrompt.includes('迁移，请在管理后台填入实际内容')) {
      const version = await prisma.promptTemplateVersion.create({
        data: {
          templateId: template.id,
          versionNo: (latestVersion?.versionNo ?? 0) + 1,
          systemPrompt: slot.systemPrompt,
          userPrompt: slot.userPrompt,
          changeNote: latestVersion ? '升级为可执行基线模板' : '种子初始化基线版本',
          isBaseline: true,
        }
      });

      await prisma.promptTemplate.update({
        where: { id: template.id },
        data: { activeVersionId: version.id }
      });
    }
  }

  console.log(`✅ 提示词模板: ${PROMPT_TEMPLATE_BASELINES.length} 个模板槽（含可执行基线版本）`);
  console.log(`   ℹ  可在 /admin/knowledge-base/prompt-templates 中基于基线版本继续微调并发布新版本`);
}

## 六、常见问题

**Q: 种子脚本执行报错 "Unique constraint failed"**  
A: 检查是否已运行过种子脚本。脚本使用 `upsert` 和 `skipDuplicates: true`，重复执行安全。若仍报错，检查数据库中是否有不符合 `@unique` 约束的旧数据需要清理。

**Q: 提示词模板页显示 `{bookTitle}`、`{analysisRules}` 这类占位符**  
A: 正常。这些是运行时替换字段，预览功能会注入样例值，正式解析任务会注入真实上下文。编辑模板时请保留需要动态替换的占位符。

**Q: 复姓无法正确提取（如"欧阳"被提取为单姓"欧"）**  
A: 检查 `surname_entries` 表中"欧阳"的 `is_compound` 字段是否为 `true`，`priority` 是否 ≥ 10。若正确但还不生效，说明 Phase 9（流水线接入）尚未完成，`extractSurname()` 仍使用硬编码列表。

**Q: 解析时发现新的角色别名不在知识库中**  
A: 在 `/admin/knowledge-base/alias-packs` 找到对应书籍类型的知识包，手动添加条目并审核通过。或者使用 LLM 生成功能批量补充。

**Q: 如何验证某条 NER 规则是否正确注入了提示词**  
A: 访问 `/admin/knowledge-base/ner-rules`，使用底部的"合并预览"功能，选择对应书籍类型，点击【预览注入列表】查看完整规则文本。
