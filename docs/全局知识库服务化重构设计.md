### 1.4 核心问题（已确认处置方案）

1. **双轨不一致** → **D2 已确认**: 硬编码全部删除，DB 为唯一数据源。`classical-names.ts` 整个文件删除，`buildAliasLookup()` 废弃，统一用 `buildAliasLookupFromDb()`
2. **6 类词表未迁移** → **D2 已确认**: hardBlock/softBlock/titleStems/positionStems 全部迁入 `ExtractionRule` 表，硬编码删除
3. **缺少关键知识类型** → 新增 `HistoricalFigureEntry`（已知历史人物标记库，非简单黑名单 — D13）、`RelationalTermEntry`、`NamePatternRule` 三张表

5. **RULIN_NAMES 数据错误** → **D1 已确认**: 牛布衣与牛浦郎是两个独立人物。`牛布衣` aliases 移除 `牛浦郎` 及牛浦相关错误映射；新增独立条目 `牛浦郎` aliases `["牛浦"]`，并以 `AliasMapping(source=IMPERSONATION)` 记录其冒用牛布衣身份的阶段性称谓

## 3. 数据模型变更

### 3.1 新增表: `historical_figure_entries`

```prisma
model HistoricalFigureEntry {
  id          String    @id @default(uuid()) @db.Uuid
  name        String    @db.VarChar(50)      // 朱元璋
  aliases     String[]  @default([])          // [太祖, 洪武爷]
  category    String    @db.VarChar(30)       // EMPEROR / SAGE / POET / GENERAL / MYTHICAL / STATESMAN
  era         String?   @db.VarChar(30)       // 明朝, 唐朝
  bookTypeId  String?   @db.Uuid             // null = 全局生效
  isActive    Boolean   @default(true)
  source      String    @default("MANUAL") @db.VarChar(20)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  bookType    BookType? @relation(fields: [bookTypeId], references: [id])

  @@unique([name, bookTypeId])
  @@map("historical_figure_entries")
}
```

**初始数据量**: ~500 条（覆盖中国主要历史人物，分 EMPEROR / SAGE / POET / GENERAL / MYTHICAL / STATESMAN 6 类）。
**数据产出方式**: LLM 按朝代/类别批量生成 → 人工抽检 → 修正提示词 → 再生成 → 入库审核（D5 已确认）。

> **D13 决策**: 此表不是"黑名单"，而是"已知历史人物标记库"。命中不意味着直接过滤，而是触发进一步判断：
> - 书内有实际经历/行为/对话/事件 → **保留**，主实体存真名，书内称谓记为 alias
> - 仅纯提及/引用/典故/背景提及 → **不提取为主角色**

### 3.3 新增表: `name_pattern_rules`

```prisma
model NamePatternRule {
  id          String    @id @default(uuid()) @db.Uuid
  pattern     String    @db.VarChar(200)     // 正则表达式
  ruleType    String    @db.VarChar(30)       // DESCRIPTIVE_PHRASE / FAMILY_HOUSE / TOO_LONG / RELATIONAL_COMPOUND
  action      String    @db.VarChar(20)       // BLOCK / WARN / ALLOW
  description String?   @db.VarChar(200)
  priority    Int       @default(0)
  isActive    Boolean   @default(true)
  bookTypeId  String?   @db.Uuid
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  bookType    BookType? @relation(fields: [bookTypeId], references: [id])

  @@map("name_pattern_rules")
}
```

**初始规则**:

| ruleType | pattern | action | 说明 |
|----------|---------|--------|------|
| FAMILY_HOUSE | `^[\u4e00-\u9fa5]{1,2}[家府]$` | BLOCK | X家/X府 |
| DESCRIPTIVE_PHRASE | `的` (name.length > 3) | BLOCK | 含"的"的长名 |
| DESCRIPTIVE_PHRASE | `之` (name.length > 3) | BLOCK | 含"之"的长名 |
| RELATIONAL_COMPOUND | `^[\u4e00-\u9fa5]{2,4}(老妻\|浑家\|丈母\|...)$` | BLOCK | 人名+亲属词 |
| DESCRIPTIVE_PHRASE | `其余` | BLOCK | 含"其余" |

## 4. 运行时加载服务重构

### 4.1 现有 `loadAnalysisRuntimeConfig()` 扩展

**文件**: `src/server/modules/knowledge/load-book-knowledge.ts`

当前加载: genericTitles + surnames + extractionRules。

**扩展为**:

```typescript
export interface FullRuntimeKnowledge {
  // 已有
  lexiconConfig: BookLexiconConfig;         // 泛称、姓氏、规则
  aliasLookup: Map<string, string>;          // 知识库别名映射

  // 新增
  historicalFigures: Set<string>;            // 历史人物名 + 别名展平
  relationalTerms: Set<string>;              // 关系词集合
  namePatternRules: NamePatternRuleCompiled[];  // 编译后的正则规则
  hardBlockSuffixes: Set<string>;            // 从 DB 加载
  softBlockSuffixes: Set<string>;            // 从 DB 加载
  titlePattern: RegExp;                      // 从 DB 加载的 stems 编译
  positionPattern: RegExp;                   // 从 DB 加载的 stems 编译
}

export async function loadFullRuntimeKnowledge(
  bookId: string,
  bookTypeKey: string | null,
  prisma: PrismaClient
): Promise<FullRuntimeKnowledge> {
  // 1. 加载基础 lexiconConfig (已有)
  // 2. 加载 aliasLookup (已有)
  // 3. 加载 historicalFigures
  //    - 全局 (bookTypeId=null) + 书籍类型级 (bookTypeId=thisType)
  //    - name + aliases 展平到 Set
  // 4. 加载 relationalTerms
  //    - 全局 + 书籍类型级
  // 5. 加载 namePatternRules
  //    - 全局 + 书籍类型级, 编译正则
  // 6. 加载 hardBlock/softBlock/titleStems/positionStems
  //    - 从 ExtractionRule 表 ruleType 过滤
  // 7. 合并 + 缓存
  return { ... };
}
```

### 4.2 缓存策略

```typescript
// 全书级缓存 — 解析开始时加载一次，传入整个 pipeline
const knowledgeCache = new Map<string, FullRuntimeKnowledge>();

export async function getOrLoadKnowledge(
  bookId: string,
  bookTypeKey: string | null,
  prisma: PrismaClient
): Promise<FullRuntimeKnowledge> {
  const cacheKey = `${bookId}:${bookTypeKey ?? "default"}`;
  if (knowledgeCache.has(cacheKey)) return knowledgeCache.get(cacheKey)!;
  const knowledge = await loadFullRuntimeKnowledge(bookId, bookTypeKey, prisma);
  knowledgeCache.set(cacheKey, knowledge);
  return knowledge;
}
```
