# Wen-Yuan（文渊）MVP 产品方案（完整版）

## 0. 文档信息
- 版本：v1.0
- 日期：2026-03-04
- 适用范围：MVP（个人使用 + 开源协作）
- 首个样本作品：《儒林外史》

## 1. 项目定位
### 1.1 项目名称
Wen-Yuan（文渊）

### 1.2 一句话定位
一个面向中国古代文学作品的 AI 人物分析工具，支持从文本导入到人物关系、时间线与人工校对的完整闭环。

### 1.3 愿景
以《儒林外史》为起点，形成“AI 自动抽取 + 人工校验 + 可视化展示”的通用能力，后续可平滑扩展到《红楼梦》《三国演义》《西游记》《水浒传》等。

### 1.4 MVP 原则
- 小而精：只做高价值闭环，不做大而全后台系统。
- 证据优先：所有关键结论必须能定位原文证据。
- 人机协同：AI 负责效率，人工负责定稿。
- 可维护：模块边界清晰，方便开源协作。

## 2. 用户与核心场景
### 2.1 用户角色
1. 主用户（你自己）
- 导入文本、触发分析、校对结果、导出研究材料。

2. 开源协作者
- 复现流程，补充规则，优化模型与提示词。

3. 文学爱好者（次级）
- 只读浏览人物关系网与时间线。

### 2.2 核心场景
1. 导入作品文本并生成版本。
2. 自动抽取人物、别名、关系、事件。
3. 浏览人物关系网与人物时间线。
4. 对错误识别进行人工修正与回滚。
5. 导出 JSON/Markdown/图谱快照。

## 3. MVP 功能全景
- F1 作品工作台（Book Workspace）
- F2 导入与预处理（Ingestion & Parsing）
- F3 人物关系网（Character Graph）
- F4 人物时间轴（Character Timeline）
- F5 人工校对中心（Review Center）
- F6 导出与复用（Export）

---

## 4. 功能详细方案（逐模块）

## 4.1 F1 作品工作台（Book Workspace）
### 4.1.1 目标
统一管理作品、版本与分析进度，提供全局入口。

### 4.1.2 页面与交互
- 页面：`/books`
- 组件：搜索栏、标签筛选、排序控件、作品卡片网格、状态过滤器。
- 卡片字段：标题、作者、朝代、标签、当前版本、状态、更新时间、错误标记。

### 4.1.3 状态机
`NOT_IMPORTED -> IMPORTED -> ANALYSIS_PENDING -> ANALYZING -> REVIEW_PENDING -> COMPLETED`
失败分支：任一状态可进入 `FAILED`。

### 4.1.4 业务规则
- 每个作品至少有 1 个版本才能分析。
- 当前“激活版本”用于关系图和时间轴默认展示。
- 有失败任务时卡片展示“失败原因入口”。

### 4.1.5 验收标准
- 可新增/编辑/归档作品。
- 支持按状态、更新时间、标签过滤。
- 作品状态与任务状态一致。

### 4.1.6 技术实现要点
- 列表查询支持分页与组合筛选。
- 状态由任务引擎驱动，避免前端自行推断。
- 列表数据短 TTL 缓存（30~60 秒）。

---

## 4.2 F2 导入与预处理（Ingestion & Parsing）
### 4.2.1 目标
稳定把原始文本转成可分析结构（章节、段落、偏移量）。

### 4.2.2 输入格式（MVP）
- `txt`
- `markdown`
- 手动粘贴
- 备注：`epub` 放 v0.3

### 4.2.3 处理流程
1. 上传文本并记录原始文件。
2. 编码识别与转换（UTF-8/GBK）。
3. 清洗（页眉页脚、重复空行、注释噪音）。
4. 章节切分（“第X回/章”规则 + 兜底分块）。
5. 段落切分并写入字符偏移。
6. 生成导入版本（version_no++）。

### 4.2.4 关键规则
- 原文不可变：原始文本只增不改。
- 版本可追溯：每次导入形成独立 `book_version`。
- 偏移可定位：任意证据可回到段落原文位置。

### 4.2.5 异常处理
- 编码失败：提示“文本编码无法识别”。
- 章节切分失败：自动回退到固定长度分块并标记 `LOW_CONFIDENCE_SEGMENT`。

### 4.2.6 验收标准
- 导入成功率 >= 98%。
- 章节切分准确率 >= 95%（抽样核验）。
- 可查看每次导入版本日志。

### 4.2.7 技术实现要点
- 大文本走异步任务，不阻塞请求。
- 计算文本 `checksum` 去重，避免重复导入浪费成本。
- 中间产物（cleaned/chunked）可缓存到对象存储。

---

## 4.3 F3 人物关系网（Character Graph）
### 4.3.1 目标
可视化人物关系并支持证据追溯。

### 4.3.2 图谱模型
- 节点：人物主名、别名数、身份标签、出场频次、置信度。
- 边：关系类型、方向、强度、置信度、首次章节、证据数量。

### 4.3.3 前端交互
- 过滤器：关系类型、章节区间、最小置信度。
- 点击节点：展示人物卡 + 出场证据。
- 点击边：展示关系判定依据（引用原文 + AI 解释）。
- 聚焦模式：仅展示选中人物 1~2 跳邻居。

### 4.3.4 业务规则
- 低置信度边默认隐藏（可手动开启）。
- 无证据边禁止进入“已确认”状态。

### 4.3.5 验收标准
- 图谱加载时延可接受（小型作品 < 2s）。
- 每条展示边至少可回查 1 条证据。
- 支持按章节范围动态刷新图谱。

### 4.3.6 技术实现要点
- 查询层先从关系表筛选，再批量拉证据摘要。
- 图谱响应采用轻量 payload（node/edge + summary），详情二次拉取。
- 热点图谱按 `book_version + filter_hash` 做短缓存。

---

## 4.4 F4 人物时间轴（Character Timeline）
### 4.4.1 目标
按叙事顺序展示人物关键事件演化。

### 4.4.2 排序策略
- 主排序：章节序号。
- 辅排序：相对时间提示词（次日、后数月、翌日等）。

### 4.4.3 事件字段
- 事件标题（自动生成，可编辑）
- 事件类型（登场/冲突/任官/迁徙/交往等）
- 章节锚点
- 参与人物
- temporal_hint
- evidence_ids
- confidence

### 4.4.4 交互
- 人物筛选：按单人物或多人对比。
- 事件筛选：按事件类型与置信度。
- 点击事件：展开证据段落与上下文。

### 4.4.5 验收标准
- 每个事件都带证据入口。
- 时间轴顺序符合章节叙事。
- 人工修正后顺序可重排并即时生效。

### 4.4.6 技术实现要点
- 时间轴查询分“列表”和“详情”两个接口。
- 事件更新后触发 timeline 缓存失效。

---

## 4.5 F5 人工校对中心（Review Center）
### 4.5.1 目标
保证结果准确与可解释，是 MVP 成败关键。

### 4.5.2 校对任务来源
- 低置信度结论。
- 冲突结论（同名异人、关系互斥、时间冲突）。
- 无证据或证据冲突结论。

### 4.5.3 校对动作
1. 人物合并（merge）
2. 人物拆分（split）
3. 别名维护（add/remove alias）
4. 关系修正（type/direction/strength）
5. 事件修正（title/participants/order）
6. 回滚（rollback by action id）

### 4.5.4 审计要求
- 记录：`before_json / after_json / reason / operator / created_at`
- 所有关键改动可回放、可撤销。

### 4.5.5 验收标准
- 合并人物后关系与事件挂接正确迁移。
- 回滚后数据可恢复到目标快照。
- 所有动作都有审计日志。

### 4.5.6 技术实现要点
- 多表改动必须事务提交。
- 合并与拆分需具备幂等保护。
- 校对动作完成后触发相关缓存失效与衍生重算。

---

## 4.6 F6 导出与复用（Export）
### 4.6.1 目标
输出可复现、可分享、可二次分析的研究材料。

### 4.6.2 导出类型
- JSON（结构化全量）
- Markdown（可读报告）
- 图谱快照（PNG/SVG）

### 4.6.3 导出内容要求
- 作品基础信息
- 分析版本信息
- 人物/关系/事件主体数据
- 每条结论对应 evidence id
- 模型与提示词版本（可选）

### 4.6.4 验收标准
- 导出文件可被再次导入或解析。
- 导出内容与当前激活版本一致。

### 4.6.5 技术实现要点
- 导出任务异步化，支持进度轮询。
- 大文件导出存对象存储并返回下载链接。

---

## 5. 全局非功能需求
### 5.1 可解释性（强约束）
- 关系和事件必须可追溯到段落级证据。
- UI 默认展示证据入口。

### 5.2 准确性
- 低置信结论默认不展示为“确定事实”。
- 人工确认优先级高于自动结果。

### 5.3 成本控制
- 分层模型调用：规则召回 -> 轻模型筛选 -> 重模型判定。
- 记录 token 和耗时，支持成本看板。

### 5.4 可扩展性
- 面向作品扩展设计，不绑定单一小说。
- 模型与规则可插件化替换。

### 5.5 可开源性
- 清晰文档、样例数据、可一键本地部署。

## 6. 核心指标（MVP）
- 导入成功率 >= 98%
- 人物抽取召回率 >= 85%
- 关系证据可追溯率 = 100%
- 校对动作成功率 >= 99%
- 任务失败可重试恢复率 >= 95%

## 7. 版本路线图
### v0.1（2~4 周）
- 作品工作台
- 导入与切分
- 人物抽取与基础关系图
- 人工校对（人物合并/关系修正）

### v0.2
- 事件抽取与时间轴
- 证据可视化增强
- 导出 JSON/Markdown

### v0.3
- 多作品对比
- 可选 Neo4j 增强查询
- 开源贡献流程完善

## 8. 风险与应对
1. 古文语言导致抽取偏差
- 应对：规则库 + LLM + 人工校验三层兜底。

2. 同名异人误合并
- 应对：上下文窗口 + 章节位置 + 身份标签冲突提示。

3. 成本波动
- 应对：模型分级调用和缓存中间结果。

4. 可视化误导
- 应对：默认高置信展示，低置信需手动开启。

## 9. README 可直接使用文案
Wen-Yuan（文渊）是一个面向中国古代文学作品的人物分析工具。项目以《儒林外史》为首个落地样本，通过 AI 自动抽取与人工校对结合的方式，构建可解释、可追溯的人物关系网络与事件时间线。项目定位为个人研究与开源协作，强调小而精、可扩展、低维护成本。
