# Wen-Yuan PRD（MVP）

## 0. 文档信息
- 版本：v1.0（MVP 工程化版）
- 更新时间：2026-03-04
- 适用阶段：v0.1 ~ v0.3
- 目标读者：产品、后端、前端、AI 工程、开源协作者

## 1. 产品定位
### 1.1 项目名称
Wen-Yuan（文渊）

### 1.2 一句话定位
一个面向中国古代文学作品的 AI 人物分析工具，支持从文本导入到人物关系、时间线与人工校对的完整闭环。

### 1.3 MVP 愿景
- 以《儒林外史》跑通“可导入、可分析、可校对、可导出”的最小闭环。
- 抽象为可扩展流程，后续支持《红楼梦》《三国演义》《西游记》《水浒传》等。
- 以个人使用和开源协作为主，不做重型企业平台。

### 1.4 明确边界（MVP 不做）
- 不做复杂多租户计费系统。
- 不做企业级组织审批流。
- 不做高并发公网商业 SLA 承诺。
- 不做全自动“零人工”结论发布，所有核心结论必须可校对。

## 2. 用户与场景
### 2.1 目标用户
- 主用户：个人研究者（你自己）。
- 次用户：开源协作者（贡献规则、模型适配、数据修正）。
- 轻量用户：文学爱好者（只读探索）。

### 2.2 核心场景
1. 导入一本古典小说文本（txt/markdown/粘贴）。
2. 自动抽取人物、别名、关系、事件。
3. 查看人物关系网与人物时间轴。
4. 对 AI 结果执行人工校对（合并/拆分/修正/回滚）。
5. 导出结构化结果（JSON/Markdown/图谱快照）。

## 3. 产品目标与成功指标
### 3.1 业务目标
- 2~4 周内上线可用 MVP，完成《儒林外史》单作品闭环。
- 人工校对后可稳定输出研究结果。

### 3.2 成功指标（MVP）
- 导入成功率 >= 98%。
- 章节切分准确率 >= 95%（人工 spot check）。
- 人物抽取召回率 >= 85%（按基准集）。
- 关系结论证据可追溯率 = 100%。
- 关键操作（合并人物/修正关系）成功率 >= 99%。
- 单书分析成本可控（按 token 和任务时长有监控）。

## 4. 功能范围（MVP）

### 4.1 模块 A：作品工作台（Book Workspace）
**目标**：管理作品、版本与分析状态。

**需求**：
- 展示作品列表卡片：标题、作者、朝代、标签、处理状态、更新时间。
- 支持搜索、排序、标签过滤。
- 支持状态流转：`未导入 -> 已导入 -> 待分析 -> 分析中 -> 待校对 -> 已完成`。

**验收标准**：
- 可创建、编辑、归档作品。
- 状态变化有时间戳且可追溯。

### 4.2 模块 B：导入与文本处理（Ingestion & Parsing）
**目标**：将文本转化为可分析结构。

**输入支持**：`txt` / `md` / 粘贴文本（epub 进入 v0.3）。

**处理流程**：
1. 文件上传与编码检测（UTF-8/GBK）。
2. 文本清洗（页眉页脚、空白、注释噪音）。
3. 章节切分（“第X回/章”规则 + 回退分块）。
4. 段落入库并记录字符 offset。
5. 生成导入版本号。

**验收标准**：
- 同一本书允许多版本导入且可切换。
- 任意段落可定位原始章节与 offset。

### 4.3 模块 C：人物关系网（Character Graph）
**目标**：可视化人物关系，且每条边可追溯证据。

**能力**：
- 节点：人物主名、别名、身份标签、出场频次。
- 边：关系类型、方向、置信度、首次章节、证据片段。
- 过滤：关系类型、置信度阈值、章节区间。

**验收标准**：
- 点击节点可查看人物卡与证据列表。
- 点击边可查看“原文引用 + AI 判定说明”。

### 4.4 模块 D：人物时间轴（Character Timeline）
**目标**：按叙事顺序展示人物关键事件。

**排序策略**：
- 主排序：章节序号。
- 辅排序：相对时间词（次日、后数月、翌日等）。

**事件字段**：
- 标题、事件类型、章节位置、时间提示、参与人物、证据片段、置信度。

**验收标准**：
- 可按人物查看时间轴。
- 每个事件都可回到原文证据。

### 4.5 模块 E：人工校对（Human-in-the-loop）
**目标**：保证准确性与可解释性。

**校对操作**：
- 人物合并、人物拆分、别名维护。
- 关系修正（类型/方向/强度）。
- 事件修正（标题/参与人/顺序）。
- 基于变更日志的一键回滚。

**验收标准**：
- 所有修改记录 `before/after/reason/operator/time`。
- 回滚后数据一致且图谱/时间轴同步刷新。

### 4.6 模块 F：导出与共享
**目标**：输出可复用研究资产。

**导出格式**：JSON、Markdown、图谱快照（PNG/SVG）。

**验收标准**：
- 导出文件包含版本号、生成时间、模型信息。
- 导出结论包含证据引用 ID。

## 5. 非功能需求
### 5.1 可解释性（强约束）
- 所有关系/事件必须绑定至少 1 条证据片段。
- UI 层默认展示证据入口。

### 5.2 准确性
- 低置信度结果默认标为“待校对”，不标注为确定事实。
- 人工修正优先级高于 AI 自动结果。

### 5.3 可扩展性
- 模型可替换（OpenAI / 本地模型）。
- 单作品闭环完成后，可平滑扩展多作品。

### 5.4 成本与性能
- 支持异步批处理与缓存中间结果。
- 记录 token、耗时、失败重试次数。

### 5.5 开源友好
- Docker Compose 一键启动。
- 样例数据与贡献指南可复现。

## 6. 关键业务规则
1. AI 结果默认状态为 `DRAFT`，人工确认后才进入 `VERIFIED`。
2. 任意“结论对象”（关系/事件）缺证据时，不允许标记为 `VERIFIED`。
3. 人物主名唯一性在“作品版本”范围内校验，不做全平台强唯一。
4. 回滚操作必须写审计日志，不允许静默覆盖。

## 7. MVP 版本路线图
### v0.1（2~4 周）
- 作品工作台。
- 导入/清洗/章节切分。
- 人物抽取 + 基础关系图。
- 基础校对（人物合并、改名）。

### v0.2
- 事件抽取 + 人物时间轴。
- 关系证据可视化增强。
- JSON/Markdown 导出。

### v0.3
- 多作品管理与对比。
- 图谱能力增强（可选 Neo4j）。
- 开源化文档与插件接口。

## 8. 风险与应对
- 古文/白话混杂导致识别偏差。
  - 应对：规则召回 + LLM 判定 + 人工兜底。
- 同名异人误合并。
  - 应对：上下文窗口 + 身份标签 + 冲突提示。
- 成本波动。
  - 应对：分层模型策略（轻模型筛选，重模型复核）。
- 可视化误导。
  - 应对：默认仅展示高置信边，低置信需手动打开。

## 9. README 可复用文案
Wen-Yuan（文渊）是一个面向中国古代文学作品的人物分析工具。项目以《儒林外史》为首个落地样本，通过“AI 自动抽取 + 人工校对”的方式，生成可解释、可追溯的人物关系网络与事件时间线。项目定位为个人研究与开源协作，强调小而精、可扩展、低维护成本。
