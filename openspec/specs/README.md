# OpenSpec 业务规范

本目录存放 Wen-Yuan 项目的业务/功能规范。

## 目录结构

- `domain/` - 业务领域模型与概念定义
- `features/` - 功能需求规范
- `constraints/` - 产品约束与边界

## 与 Trellis 的关系

- **OpenSpec**: 定义"做什么"（业务需求、功能设计）
- **Trellis**: 定义"怎么做"（技术标准、代码规范）

## 规范原则

1. 业务优先：以业务价值和用户需求为导向
2. 可验证性：需求必须可测试、可验证
3. 可追溯性：功能变更必须关联到业务需求
4. 演进式：规范随业务发展持续演进
